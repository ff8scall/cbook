# 🗺️ 전략 지도 (STRATEGY_MAP.md)

## 1. 프로젝트 개요
- **프로젝트명**: 만화책 할인 통합 뷰어 (개인 목적)
- **핵심 가치**: 분산된 만화책 단행본 할인 정보를 한눈에 파악하여, 개인의 지출 효율성을 극대화하고 구매 타이밍을 놓치지 않음.

## 2. 시스템 아키텍처 전략 (Zero-Cost Serverless)
개인 사용 목적이므로 서버 유지비용을 0원으로 수렴하게 만드는 아키텍처를 채택함.

- **Data Collector (스크래퍼)**: GitHub Actions
  - 지정된 시간(하루 2회)에 Node.js 스크립트 실행.
  - 3대 플랫폼 데이터 수집 후 정규화하여 `data.json` 형태로 생성.
  - 이 JSON 파일을 저장소에 자동 커밋&푸시.
- **Frontend (웹 뷰어)**: Vercel 또는 GitHub Pages (React/Next.js 기반)
  - GitHub 저장소에 `data.json`이 업데이트될 때마다 원격 JSON을 fetch하여 화면에 렌더링.
- **DB**: 별도의 RDBMS 없이 JSON 파일 기반 정적 데이터 운영.

## 3. UI/UX 전략
- **디자인 톤앤매너**: 눈이 편안한 Dark Mode 기본 적용 (책 표지가 돋보이는 갤러리 뷰).
- **인터랙션**: 무한 스크롤 혹은 그리드 뷰, 카드 호버 시 해당 플랫폼 구매 페이지 바로가기 버튼 노출.

## 4. 리스크 및 방어 전략 (Pre-Verify)
- **차단(Ban) 리스크**: 스크래핑 주기를 최소화(12시간 단위)하고 최신 브라우저 헤더로 위장하여 방어.
- **플랫폼 구조 변경**: 각 플랫폼의 스크래퍼를 독립적인 모듈로 분리(Adapter 패턴). 하나의 스크래퍼가 고장 나도 전체 시스템 장애로 이어지지 않음.
